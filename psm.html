4. Diagnostico

Asumiendo un escenario con dos niveles de exposiciÃ³n: - Expuesto \(X = 1\) - No expuesto \(X=0\)

De tal forma que, los resultados potenciales son: - Con exposiciÃ³n (\(Y(1) = [Y_1|X=1]\)) - Sin exposiciÃ³n (\(Y(0) = [Y_0|X=0]\)).

Entonces, la diferencia entre estos resultados para comprender el impacto de la exposiciÃ³n en el resultado es \(Y(1) - Y(0)\).

Los supuestos son:

  1. Consistencia
  2. Intercambiabilidad
  3. Positividad

Estos supuestos se denominan a veces condiciones de identificabilidad, ya que necesitamos que se cumplan para identificar las estimaciones causales.

Consistencia

  1. Consistencia: Suponemos que la pregunta causal que afirma responder es coherente con la que realmente responde el anÃ¡lisis. MatemÃ¡ticamente, esto significa: \[ Y_\texttt{obs} = (X)[Y(1)] + (1-X)[Y(0)]. \]

En otras palabras, el resultado observado es exactamente igual al resultado potencial bajo la exposiciÃ³n recibida. Dos formas habituales de hablar de este supuesto son:

  • Consistencia con exposiciÃ³n bien definida: Suponemos que para cada valor de la exposiciÃ³n, no hay diferencia entre los sujetos en la asignaciÃ³n de esa exposiciÃ³n. Dicho de otro modo, no existen mÃºltiples versiones del tratamiento.

  • Consistencia sin interferencias: Suponemos que el resultado (tÃ©cnicamente todos los resultados potenciales, independientemente de si se observan o no) para cualquier sujeto no depende de la exposiciÃ³n de otro sujeto.

Este supuesto a veces se denomina como supuesto de valor de tratamiento unitario estable (stable-unit-treatment-value-assumption) o SUTVA (Imbens y Rubin, 2015).

Intercambiabilidad

Suponemos que dentro de los niveles de las variables relevantes o factores de confusiÃ³n, los sujetos expuestos y no expuestos tienen la misma probabilidad de experimentar cualquier resultado antes de la exposiciÃ³n; es decir, los sujetos expuestos y no expuestos son intercambiables.

Este supuesto a veces se denomina ausencia de factores de confusiÃ³n no medidos.

Positividad

Suponemos que dentro de cada nivel y combinaciÃ³n de las variables de estudio utilizadas para lograr la intercambiabilidad, hay sujetos expuestos y no expuestos. Dicho de otro modo, cada individuo tiene alguna posibilidad de experimentar cada nivel de exposiciÃ³n disponible. A veces, esto se denomina suposiciÃ³n probabilÃ­stica.

Ponderar

Es comÃºn en estos casos observar desbalance en la distribuciÃ³n de los scores entre el grupo de expuestos y no expuestos. Esto es por los conteos mÃ¡s altos del grupo de no expuestos.

La ponderaciÃ³n permite que el score sea suave, aplicando una transformaciÃ³n que permita, en promedio, que las covariables de interÃ©s estÃ©n equilibradas en la poblaciÃ³n ponderada. Para ello, construiremos un peso utilizando el score de propensiÃ³n.

ATT puede estimarse mediante ponderaciÃ³n. La ponderaciÃ³n se estima de la forma:

\[ w_{ATT} = X - \frac{(1-X) \cdot p}{(1-p)} \]

Es comÃºn usar ATT cuando se utiliza matching, todas las observaciones expuestas se incluyen y se âemparejanâ con las observaciones de control, algunas de las cuales pueden descartarse.

6. EstimaciÃ³n de varianza

Una forma de estimar la varianza del estimador es mediante Bootstrap Sampling. Bootstrap es un algoritmo sencillo para calcular estimaciones o estadÃ­sticos medianteremuestreo con sustituciÃ³n.

Resulta Ãºtil cuando no existe una soluciÃ³n de forma cerrada para calcular algo, como suele ocurrir en la inferencia causal (sobre todo para los errores estÃ¡ndar), y cuando sospechamos que los cÃ¡lculos paramÃ©tricos no son vÃ¡lidos para una situaciÃ³n determinada.

El proceso consiste en:

  1. Re muestrear los datos con reemplazo
  2. Ajustar el estimador a la muestra Bootstrap
  3. Repite R veces.

Finalmente, se obtiene la distribuciÃ³n de estimaciones, con la que podemos calcular estadÃ­sticos descriptivos de poblaciÃ³n, como estimaciones puntuales, errores estÃ¡ndar e intervalos de confianza.

# FunciÃ³n de workflow
fit_psm <- function(split) {
  # convierte muestra bootstrapped a df
  .df <- analysis(split)

  formula_net <-  net ~ income + health + temperature + genetic_resistance
  # Estimas scores
  p_model <- glm(
    formula_net,
    data = .df,
    family = binomial()
  )

  # Calculo de pesos
  .df <- p_model |>
    augment(type.predict = "response", data = .df) |>
    mutate(wts = wt_ate(.fitted, net))

  # Match de hogares
  matched <- matchit(
    formula_net,
    data = .df,
    distance = "glm",
    link = "logit", 
    s.weights = .df$wts,
    replace = T
  )
  .df <- get_matches(matched)
  
  # EstimaciÃ³n de efecto
  lm(malaria_risk ~ net, data = .df, weights = wts) |>
    tidy()
}
